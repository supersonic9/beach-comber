# Review 
Review this project's plan in the SCOPE.md file as a staff engineer. Challenge assumptions, identify risks, suggest alternatives. Call out any major gaps, incorrect assumptions, or areas that need more detail. Be critical but constructive — the goal is to improve the plan, not just criticize it. 

For any risks identified or gaps, list possible solutions along with a recommendation. 

Provide actionable feedback that the team can use to strengthen the plan before implementation begins. Also point out any areas that need to be clarified before Claude Code can confidently implement the plan. If and once you are satisfied the plan is solid and clear then mark the plan as approved.
